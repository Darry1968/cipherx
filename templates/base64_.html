What is Base 64?

Base64 is a binary-to-text encoding scheme that represents binary data in an ASCII string format by translating it into a radix-64 representation. The term "Base64" originates from the fact that it uses a 64-character alphabet, which consists of the upper and lowercase letters A-Z, the numbers 0-9, and the symbols "+" and "/".
Base64 is not a cryptographic cipher in the traditional sense, as it does not provide any encryption or decryption functionality. However, it is often used with other cryptographic ciphers, such as RSA, to encode and transmit data securely.

3.1 Encoding Process
1. Divide the Input Data:
- Break the binary data into chunks of 3 bytes (24 bits).
2. Convert to 4 ASCII Characters:
- Convert each 24-bit chunk into a set of four 6-bit values. 3. Map to Base64 Characters:
- Map each 6-bit value to its corresponding Base64 character. 4. Padding (if necessary):
- Add padding with '=' characters if the input data's length is not a multiple of 3 bytes.

# 3.2 Example
Consider the string "Hello" in ASCII:
- ASCII Binary Representation: `01001000 01100101 01101100 01101100 01101111` - Grouped in 3 Bytes: `01001000 01100101 01101100` and `01101100 01101111` Converted to Base64:
1. `010010` `000110` `010101` `100110` `110110` `110011` `011011` `110011` `011011` `110111`
2. Encoded as Base64: `SGVsbG8=`

5. Use Case in a Real-World Scenario

# 5.1 Email Attachments
- Scenario: Sending an Image via Email
- Background: Alice wants to send a picture to Bob via email. - Implementation: The image is converted to Base64 encoding. The Base64-encoded text is embedded into the email body. Bob's email client decodes the Base64 text to reconstruct the image.

# 5.2 Data Storage in JSON
- Scenario: Storing Binary Data in JSON
- Background: A web application needs to store binary data in a JSON object. - Implementation: The binary data is Base64 encoded. The Base64-encoded string is included as a field in the JSON object. The client-side application decodes the Base64 string to retrieve the original binary data.

# 5.3 URL Data
- Scenario: Embedding Binary Data in URLs
- Background: A web application needs to include binary data in a URL. - Implementation: The binary data is Base64 encoded. The Base64-encoded string is added to the URL. The server-side application decodes the Base64 string to retrieve the original binary data.
